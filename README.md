## Methodology: 8 Principles

### 1. Architecture First
Define the system shape before writing implementation. The file tree, command set, and data flow come before code.

### 2. Trust Deeds
Every system needs governance. The trust deed defines what can execute, what cannot, and what requires verification. [TRUST_DEED.md](TRUST_DEED.md) governs this cockpit.

### 3. Verification Surfaces
SHA-256 seals, Ada contract gates, Prolog rule checks, envelope verification — multiple independent verification layers ensure integrity.

### 4. Agentic Product Compression
Prior architecture + reusable constitutions + repeatable prompts + repo patterns + LLM implementation agents = **high-velocity product spawning**.

### 5. Repo as Runtime
The GitHub repository is not passive storage. It is the active manifest, proof source, and cold-boot substrate.

### 6. Audit Everything
Every command, every agent run, every capsule creation, every mode switch writes to Woz Vault. Nothing is silent.

### 7. Simulate Before Implementing
All VM execution, Ada contracts, Lisp evaluation, Ollama inference, and Brainfuck tape running are browser-safe simulations. Real runtimes replace simulations incrementally.

### 8. Replace Simulations Incrementally
JS simulation → WASM compilation → native execution. Each layer can be swapped without changing the trust architecture.
